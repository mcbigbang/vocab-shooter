# Vocab Shooter - 英语词汇射击游戏

## 1. 项目概述

**项目名称**: Vocab Shooter（词汇射击）
**类型**: 实时互动教学游戏
**核心功能**: 词汇下落点击射击，师生同屏实时互动
**目标用户**: 在线英语教师与低龄初学者

---

## 2. 功能设计

### 2.1 角色划分

| 角色 | 操作端 | 说明 |
|------|--------|------|
| 教师 | PC浏览器 | 主持游戏，监控得分，实时互动 |
| 学生 | PC/iPad浏览器 | 点击射击下落词汇 |

### 2.2 游戏流程

**第一步：创建房间（教师操作）**
- 输入词汇列表（每行一个，或粘贴词表）
- 设置游戏参数（下落速度、干扰词数量）
- 生成房间码（或直接分享链接）

**第二步：学生加入（学生操作）**
- 点击链接进入房间
- 无需注册，输入昵称即可

**第三步：开始游戏**
- 教师点击"开始"
- 学生屏幕：词汇从顶部缓缓下落
- 学生点击正确词汇 → +1分
- 学生点击错误词汇 → -1分
- 词汇落到底部消失 → 不扣分

**第四步：实时监控**
- 教师屏幕同步显示学生得分、命中数、错误数
- 教师可随时暂停、添加词汇、结束游戏

### 2.3 游戏参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 下落速度 | 60px/秒 | 初级阶段，慢速 |
| 干扰词比例 | 30% | 干扰词数量占总词汇的比例 |
| 同屏词汇数 | 最多5个 | 同时下落的最大词汇数 |
| 游戏时长 | 3分钟 | 可配置 |

### 2.4 词汇来源

- 教师手动输入
- 支持粘贴词表（每行一个词汇）
- 可选：复用CATTI助手的词库

## 3. 技术方案

### 3.1 架构

**实时同步方案**: WebSocket（Socket.IO）
- 无需服务器：可用独立部署或集成到CATTI助手的FastAPI

**部署方式**: 
- 集成到现有的 catti-tool FastAPI 服务
- 新增 `/vocab-shooter/` 路由和 WebSocket 端点

### 3.2 页面

| 页面 | 路径 | 说明 |
|------|------|------|
| 教师创建房间 | /vocab-shooter/host | 输入词汇，创建房间 |
| 学生加入 | /vocab-shooter/player | 输入昵称和房间码加入 |
| 游戏画面 | /vocab-shooter/game | 共享的游戏画面 |

### 3.3 项目结构

```
vocab-shooter/
├── static/
│   ├── host.html       # 教师端
│   ├── player.html     # 学生端
│   └── game.js        # 游戏核心逻辑
├── app/
│   └── shooter.py      # WebSocket处理、房间管理
└── SPEC.md
```

## 4. 游戏界面设计

### 4.1 教师端（Host）

- 左侧：词汇输入区 + 参数设置
- 右侧：实时游戏画面预览
- 底部：得分板（学生昵称、分数、命中率）
- 控制按钮：开始 / 暂停 / 添加词汇 / 结束

### 4.2 学生端（Player）

- 全屏游戏区域
- 顶部：得分显示
- 底部：生命值/时间条（可选）
- 点击词汇时有爆炸动画效果

### 4.3 视觉风格

- 卡通风格，明亮色彩
- 适合低龄儿童
- 圆润的卡片式词汇气泡
- 击中时：绿色爆炸 + 星星特效
- 打错时：红色闪烁 + 抖动效果

## 5. 评分与反馈

### 5.1 即时反馈

| 情况 | 效果 |
|------|------|
| 击对词汇 | 绿色 +1 飘字，爆炸动画 |
| 击错词汇 | 红色 -1 飘字，屏幕抖动 |
| 词汇落底 | 词汇淡出消失 |

### 5.2 实时数据（教师端显示）

- 当前分数
- 命中数 / 总击打数
- 准确率百分比
- 本局剩余时间
